# Complyr v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Complyr v1: an open source structured dataset of GCC digital-policy regulation with a two-page Astro dashboard, schema-validated JSON data, and GitHub Pages deployment at ampedfitness/complyr.

**Architecture:** Astro static site with React islands for the interactive Developments explorer and language toggle. All data lives as JSON under `/data` (one file per document record), loaded at build time by a shared data module that computes inverse relationships, derived years, and site stats. A Node validation script (Ajv) enforces the JSON Schemas plus referential integrity, run locally and in CI on every PR.

**Tech Stack:** Astro 5, React 19, TypeScript, Ajv 8 (validation), @fontsource packages (self-hosted fonts), GitHub Actions (CI + Pages deploy). No backend, no database, vanilla CSS with custom properties (no Tailwind, to keep the gazette-meets-dashboard look hand-crafted).

**Authoritative spec:** the user's build brief (in conversation; key constraints repeated inline below). Where this plan and the brief disagree, the brief wins. Writing rules apply to ALL prose in code, data, README: no em dashes or any dashes as punctuation, plain professional tone, active voice, sentence case, no signposting.

**Attribution:** author name/links are PLACEHOLDERS everywhere, marked `<!-- AUTHOR: fill in -->` or `[Author name]`. Licences: MIT (code), CC BY 4.0 (data), copyright holder placeholder.

## Task 1: taxonomy.json

Single file, four top-level keys: `classification_rules`, `instrument_classes`, `themes`, `lifecycle`, plus `binding_statuses` and `relationship_types`.

- [ ] `classification_rules`: array of the MECE rules from the brief verbatim in spirit: leaf-only tagging, parents computed, multi-tag allowed, no general/other/misc, scope-not-label tagging, boundary calls (breach duties under data protection not cybersecurity; AML under finance regardless of issuer; listing rules under capital markets not corporate law; promulgating decrees live in citation metadata not as records).
- [ ] `instrument_classes`: binding (law, decree, regulation, resolution, decision, circular, rulebook_module) and non-binding (national_strategy, policy, guideline, framework, handbook, code_of_practice, standard, consultation_paper, white_paper). Each: `id`, `label`, `label_ar`, `binding`, `scope` (with explicit out-of-scope clause naming where excluded items belong), and `jurisdiction_forms` map where form varies (e.g. law: SA nizam by royal decree, AE federal law / federal decree-law, BH/QA/KW law, OM royal decree law).
- [ ] `themes`: all 16 branches from the brief with `id`, `label`, `label_ar`, `scope`, `coverage` (full/partial/stub per brief), `children` array of leaves each with `id` (dotted: `technology_digital.data_protection_privacy`), `label`, `label_ar`, `scope`. Every node gets a real scope statement with in/out boundaries. Coverage: Technology & Digital full, Finance partial, all others stub.
- [ ] `lifecycle`: consultation, enacted, in_force, amended, repealed, superseded, each with label + scope note (superseded explicitly distinct from repealed: quiet replacement without formal repeal).
- [ ] `binding_statuses`: binding, non_binding, conditionally_binding (requires `enabled_by` relationship).
- [ ] `relationship_types`: forward/inverse pairs: amends/amended_by, repeals/repealed_by, supersedes/superseded_by, implements/implemented_by, consults_on/resulted_from_consultation, enabled_by/enables, references/referenced_by (note: use sparingly). Contributors enter forward only.
- [ ] Commit `feat: taxonomy with 16 theme branches, instrument classes, classification rules`.

## Task 2: Schemas + validation + CI

- [ ] `schema/document.schema.json` (draft 2020-12). Required: id (pattern `^(bh|sa|ae|qa|kw|om)-[a-z0-9-]+$`), title, jurisdiction (enum BH SA AE QA KW OM), instrument_class, binding_status, lifecycle, themes (minItems 1), issuing_authority, language_of_official_text (ar/en/both), date_issued (format date), summary, impact_note, source_url (format uri), source_confidence (official/secondary/pending_verification), last_verified. Nullable: title_ar, official_title(_ar), date_effective, compliance_deadline, gazette (object: name, number, date), citation, summary_ar, impact_note_ar, obligations (array of {description, obligated_party, deadline?}), applicability ({entity_types[], sectors[], extraterritorial bool, penalties, grace_period}), relationships (array {type, target_id}), tags, english_text_unofficial (bool flag for unofficial translation), notes. `additionalProperties: false`. No `year` field stored: derived at build.
- [ ] `schema/authorities.schema.json`: array of {id, name, name_ar, jurisdiction, type (ministry/sectoral_regulator/central_bank/cabinet/head_of_state/free_zone_authority), mandate, website}.
- [ ] `schema/taxonomy.schema.json`: shape check for Task 1 file.
- [ ] `scripts/validate.mjs`: Ajv strict, formats. Beyond schema: every `themes` entry is a leaf id present in taxonomy; never a branch id; every `issuing_authority` resolves; every relationship `type` is a forward type; every `target_id` resolves to an existing document id; ids unique; file name matches id; conditionally_binding requires an enabled_by relationship; only forward relationship types allowed (inverses rejected with a message naming the forward form). Exit 1 with file:field error lines.
- [ ] Write two deliberately broken fixture files under `scripts/fixtures/`, assert validate fails on them (smoke test in `scripts/validate.test.mjs`, run via `node --test`).
- [ ] `.github/workflows/ci.yml`: on push + PR: npm ci, `node --test scripts/`, `node scripts/validate.mjs`, `npm run build`.
- [ ] `package.json` scripts: `validate`, `test`, `build`, `dev`.
- [ ] Commit `feat: json schemas, validator with referential integrity, ci`.

## Task 4: Seed dataset (research-heavy)

Procedure per entry: web-verify against official source (gazette, regulator site, official portal); capture official title, dates, citation, source URL. If only secondary sources confirm, `source_confidence: secondary`; if unverifiable, `pending_verification` plus `notes` flag for author review. Arabic fields only where confident; otherwise null. Impact notes written as analyst-to-executive briefings. Obligations itemised with parties and deadlines where the instrument sets them.

Candidate list (target 25 to 30 final records; drop, do not pad, anything that fails verification):

- SA: PDPL (Royal Decree M/19, 2021); PDPL amendments (M/148, 2023); PDPL Implementing Regulations (2024); Regulation on Personal Data Transfer outside the Kingdom (2024 and amendment); SDAIA AI Ethics Principles; SDAIA Generative AI Guidelines (public + government); SDAIA Data Protection rules/guidelines as warranted; CST Cloud Computing Regulatory Framework.
- AE: Federal Decree-Law 45/2021 (PDPL); UAE Charter for the Development and Use of AI (2024); DIFC DP Law 5/2020 Regulation 10 amendment on autonomous systems (2023); ADGM DP Regulations 2021; relevant cabinet resolutions if verifiable.
- BH: PDPL Law 30/2018 as the anchor enabling law; 2022 ministerial resolutions implementing it; CBB rulebook modules (open banking / digital) if verifiable.
- QA: Law 13/2016 (PDPPL) anchor; NPC/NCGAA guidelines; QFC DP Regulations 2021; MCIT generative AI guidance if verifiable.
- KW: CITRA Data Privacy Protection Regulation (2021).
- OM: Royal Decree 6/2022 (PDPL); Executive Regulation (MD 34/2024).

- [ ] Research and write records in batches by jurisdiction; run `npm run validate` after each batch; commit per jurisdiction (`data: saudi arabia seed entries` etc.).
- [ ] Relationships entered forward-only (implementing regs `implements` their law; amendments `amend`; transfers regs `enabled_by` where conditionally binding).
- [ ] Final pass: every record has source_url + source_confidence + last_verified = today; promulgating decrees folded into citation metadata only.
